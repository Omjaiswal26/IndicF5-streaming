TTFT tuning (time to first audio chunk)

TTFT is dominated by the first diffusion pass (cfm.sample). Lower NFE steps and a smaller first chunk (max chars) reduce wait time. The UI request overrides the server --nfe-step CLI flag unless you check “use server default”.

  • NFE 8 — ~half CFM time vs 16; slight quality loss.
  • Max chars 40–60 — shorter first chunk → fewer mel frames → lower TTFT.
  • Server CLI — --nfe-step 8 only applies when UI uses server default.
